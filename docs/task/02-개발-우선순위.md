# 포커스팟 개발 우선순위

## PRD 핵심 기능 분석

PRD에서 정의된 핵심 기능 3가지:

### 1. 위치 기반 카페 마커 표시
- **설명**: 사용자 위치를 기반으로 주변 카페를 지도에 마커로 표시
- **사용자 플로우**: 메인 지도 화면 → 위치 권한 허용 → 주변 카페 핀 표시
- **Definition of Done**: 내 주변 위치 기반 카페 마커 표시 기능

### 2. 작업 환경 맞춤 필터
- **설명**: 콘센트 개수, 의자 편안함, 소음 수준 등 조건별 검색
- **사용자 플로우**: 필터 바 → 조건 선택 → 조건에 맞는 카페만 표시
- **Definition of Done**: 작업 환경 맞춤 필터(3종 이상) 정상 작동

### 3. 실시간 정보 업데이트
- **설명**: 현재 카페 이용자가 버튼 하나로 소음/혼잡 상태 업데이트
- **사용자 플로우**: 카페 상세 페이지 → '지금 여기는?' 영역 → 상태 버튼 클릭 → 즉시 업데이트
- **Definition of Done**: 버튼 터치 한 번으로 실시간 소음 상태가 반영되는 시스템

## 개발 순서 제안

### 우선순위 1: 위치 기반 카페 마커 표시 (예상 소요 시간: 3-4일)
**이유**: 
- 가장 기본적인 기능으로 다른 기능의 기반이 됨
- 사용자가 서비스를 처음 접할 때 가장 먼저 보는 화면
- 지도 API 연동 및 위치 권한 처리 필요

**작업 내용**:
- Next.js 프로젝트 초기화 및 기본 설정
- 지도 API 연동 (네이버/카카오맵)
- 위치 권한 요청 및 현재 위치 가져오기
- 카페 데이터 구조 설계 및 목업 데이터 생성
- 지도에 카페 마커 표시
- 카페 마커 클릭 시 하단 카드 표시

**예상 소요 시간**: 3-4일

---

### 우선순위 2: 작업 환경 맞춤 필터 (예상 소요 시간: 2-3일)
**이유**:
- 사용자가 원하는 카페를 빠르게 찾을 수 있게 해주는 핵심 기능
- 마커 표시 기능이 완성된 후 필터링 로직 구현이 용이

**작업 내용**:
- 필터 UI 컴포넌트 개발 (콘센트, 소음, 의자 등)
- 필터 상태 관리 (Zustand)
- 필터 조건에 따른 카페 목록 필터링 로직
- 필터 적용 시 지도 마커 업데이트

**예상 소요 시간**: 2-3일

---

### 우선순위 3: 실시간 정보 업데이트 (예상 소요 시간: 2-3일)
**이유**:
- 사용자 참여를 유도하는 핵심 기능
- 다른 기능들이 안정화된 후 구현하는 것이 좋음

**작업 내용**:
- 카페 상세 페이지 개발
- 실시간 상태 업데이트 UI 컴포넌트
- API 엔드포인트 개발 (상태 업데이트)
- 상태 업데이트 후 지도/상세 페이지 반영
- 마지막 업데이트 시간 표시

**예상 소요 시간**: 2-3일

## 전체 개발 일정

| 단계 | 기능 | 예상 소요 시간 | 누적 시간 |
|------|------|---------------|-----------|
| 1 | 위치 기반 카페 마커 표시 | 3-4일 | 3-4일 |
| 2 | 작업 환경 맞춤 필터 | 2-3일 | 5-7일 |
| 3 | 실시간 정보 업데이트 | 2-3일 | 7-10일 |

**총 예상 소요 시간**: 7-10일 (1인 개발 기준)

## 주의사항

### Nope 섹션 확인
- ❌ 사용자 간 실시간 채팅/커뮤니티 기능은 구현하지 않음
- ❌ 앱 내 결제 및 예약 기능은 구현하지 않음
- ✅ 대신: 한 줄 평 기능, 전화 걸기 버튼, 외부 예약 링크 연결 가능

### Definition of Done 체크리스트
각 기능 완료 시 다음을 확인:
- [ ] 내 주변 위치 기반 카페 마커 표시 기능
- [ ] 작업 환경 맞춤 필터(3종 이상) 정상 작동
- [ ] 버튼 터치 한 번으로 실시간 소음 상태가 반영되는 시스템

### 치명적 오류 방지
- 지도 상의 카페 위치가 실제와 50m 이상 차이 나지 않도록 주의
- 실시간 정보 업데이트가 서버에 반영되는지 확인
- 회원가입 및 위치 정보 수집에 대한 법적 고지 포함
